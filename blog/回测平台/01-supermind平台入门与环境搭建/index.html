SuperMind平台概述：专业量化交易的起点

在量化交易的世界里，选择一个功能强大、数据丰富的平台是成功的第一步。同花顺SuperMind量化交易平台作为业内领先的专业量化投资平台，为广大投资者提供了从策略研究到实盘交易的一站式解决方案。

平台核心优势

SuperMind平台的突出特点可以用"全、快、智、简"四个字来概括：

数据全面覆盖：平台提供2014年至今完整的Level-2行情数据，涵盖股票、基金、期货、可转债等多个品种。不仅有日线、分钟线数据，还提供tick级别的精细数据，为策略开发提供了充足的数据支撑。

回测极速高效：真正的秒级回测引擎让策略验证变得高效便捷。300只股票4年历史数据的回测仅需0.29秒，这种极快的回测速度让投资者能够快速迭代策略，大大提升研发效率。

智能选股支持：平台集成了同花顺强大的"i问财"自然语言选股功能，通过简单的自然语言描述就能筛选出符合条件的股票，降低了量化交易的门槛。

简单易学易用：支持广泛使用的Python语言，配合完整的API文档和丰富的示例代码，让编程基础薄弱的用户也能快速上手。

支持的交易品种

SuperMind平台目前支持七大类交易策略开发：

  1. 股票策略：适用于A股股票、场内基金、可转债的策略回测
  2. 股票日内策略：专门用于股票日内回转交易策略
  3. 期货期权策略：支持期货和期权合约的策略回测
  4. 股票期货对冲：股票与期货的对冲策略回测
  5. 场外基金策略：场外基金申赎策略回测
  6. 外汇策略：外汇合约策略回测
  7. T+D合约策略：延期交收合约策略回测

每种策略类型都有对应的专业API接口，确保策略开发的精确性和可靠性。

平台注册与环境配置

账号注册与登录

访问SuperMind官网，点击注册按钮，按照提示完成账号注册。注册成功后，你将获得一个独立的研究环境，所有的策略代码和数据都将在这个环境中运行和管理。

研究环境介绍

SuperMind为每个用户提供了独立的研究环境，这是平台安全性的重要保障：

环境隔离：每个用户的策略都部署在独立的容器中运行，策略代码、数据和运行环境完全隔离，确保策略安全。

资源分配：免费用户享有2GB内存的研究环境，足以满足大部分策略开发和回测需求。对于需要更大计算资源的用户，平台还提供付费升级选项。

数据安全：平台采用网络隔离、访问控制和加密技术等多重安全措施，确保用户策略的机密性、完整性和可用性。

策略库管理界面

登录平台后，你将看到直观友好的策略管理界面：

策略列表：展示所有创建的策略，包括策略名称、创建时间、运行状态等信息。

代码编辑器：内置的代码编辑器支持语法高亮、自动补全等功能，提供良好的编程体验。

回测中心：专门的回测管理区域，可以查看历史回测记录、绩效报告等详细信息。

数据浏览器：便捷的数据查看工具，可以快速浏览各类金融数据，为策略开发提供参考。

from mindgo_api import *

def init(context):
    """
    初始化函数，策略启动时执行一次
    用于设置策略参数、订阅股票等
    """
    # 设置要交易的股票（平安银行）
    context.stock = '000001.SZ'
    
    # 打印初始化信息
    log.info('策略初始化完成，交易标的：%s' % context.stock)

def handle_bar(context, bar_dict):
    """
    主交易函数，每个交易频率（日/分钟）自动调用一次
    在这里编写交易逻辑
    
    参数：
    context - 上下文对象，包含账户信息、策略状态等
    bar_dict - 当前时刻所有订阅股票的K线数据
    """
    # 获取当前股票的价格数据
    stock = context.stock
    
    # 打印当前价格（用于演示）
    if stock in bar_dict:
        current_price = bar_dict[stock].close
        log.info('%s 当前价格：%.2f元' % (stock, current_price))
        
        # 简单的交易逻辑：每天买入100股
        order(stock, 100)
        log.info('买入100股 %s' % stock)

代码解析

让我们详细解析这个基础策略的每个部分：

导入库文件：from mindgo_api import * 导入SuperMind提供的所有API函数，这是每个策略都必须包含的导入语句。

init()函数：这是策略的初始化函数，在策略启动时执行一次。在这里我们可以：

  • 设置要交易的股票标的
  • 配置策略参数
  • 初始化全局变量

handle_bar()函数：这是策略的核心交易函数，根据设定的交易频率（日线或分钟线）自动调用。在这个函数中，我们：

  • 获取当前的行情数据
  • 执行交易逻辑
  • 下单交易

context对象：这是策略的上下文对象，包含了账户信息、持仓情况、策略状态等重要数据。通过context对象，我们可以：

  • 获取当前账户资金
  • 查看持仓信息
  • 存储策略状态

bar_dict参数：这是一个字典对象，包含了当前时刻所有订阅股票的K线数据。每个股票代码对应一个bar对象，包含开盘价、收盘价、最高价、最低价、成交量等信息。

运行策略

完成代码编写后，点击"运行回测"按钮，设置回测参数：

  • 回测开始时间：选择2023-01-01
  • 回测结束时间：选择2023-12-31
  • 初始资金：设置为100000元
  • 交易频率：选择日线

点击"开始回测"，平台将开始执行策略。回测过程中，你可以实时查看：

日志输出：在日志窗口中可以看到策略的执行情况，包括买入卖出记录、价格信息等。

绩效指标：回测完成后，系统会自动生成详细的绩效报告，包括收益率、夏普比率、最大回撤等关键指标。

交易记录：详细的每一笔交易记录，包括成交时间、价格、数量等信息。

def handle_bar(context, bar_dict):
    stock = context.stock
    
    # 输出调试信息
    log.info('=== 新的一天开始 ===')
    log.info('当前日期：%s' % context.current_dt)
    log.info('账户现金：%.2f' % context.portfolio.cash)
    log.info('持仓市值：%.2f' % context.portfolio.positions_value)
    
    # 获取当前价格
    if stock in bar_dict:
        current_price = bar_dict[stock].close
        log.info('%s 当前价格：%.2f' % (stock, current_price))
        
        # 检查是否有足够资金
        if context.portfolio.cash >= current_price * 100:
            order(stock, 100)
            log.info('成功买入100股 %s' % stock)
        else:
            log.warning('资金不足，无法买入')
    else:
        log.error('未获取到 %s 的行情数据' % stock)

from mindgo_api import *

def init(context):
    """策略初始化"""
    # 基本参数配置
    context.stocks = ['000001.SZ', '000002.SZ', '600000.SH']  # 股票池
    context.buy_amount = 100  # 每次买入数量
    context.max_position = 0.8  # 最大仓位
    context.stop_loss = 0.05  # 止损比例
    context.take_profit = 0.15  # 止盈比例
    
    log.info('策略参数初始化完成')
    log.info('股票池：%s' % str(context.stocks))
    log.info('单次买入数量：%d' % context.buy_amount)

def handle_bar(context, bar_dict):
    """主交易逻辑"""
    for stock in context.stocks:
        if stock in bar_dict:
            current_price = bar_dict[stock].close
            
            # 检查持仓
            position = context.portfolio.positions.get(stock)
            if position:
                # 已有持仓，检查止盈止损
                cost_price = position.avg_cost
                profit_pct = (current_price - cost_price) / cost_price
                
                if profit_pct >= context.take_profit:
                    # 达到止盈，卖出
                    order_target(stock, 0)
                    log.info('%s 达到止盈，卖出清仓' % stock)
                elif profit_pct <= -context.stop_loss:
                    # 达到止损，卖出
                    order_target(stock, 0)
                    log.info('%s 达到止损，卖出清仓' % stock)
            else:
                # 没有持仓，考虑买入
                # 简单的买入逻辑：价格大于20日均线
                hist = history(stock, ['close'], 20, '1d')
                if len(hist) >= 20:
                    ma20 = hist['close'].mean()
                    if current_price > ma20:
                        # 买入逻辑
                        order(stock, context.buy_amount)
                        log.info('%s 价格突破20日均线，买入%d股' % (stock, context.buy_amount))

这个进阶示例展示了如何将策略参数化，并加入了一些基本的交易逻辑，包括：

多股票管理：同时管理多只股票，每只股票独立判断。

仓位控制：设置最大仓位限制，避免过度集中风险。

止盈止损：为每笔交易设置明确的止盈止损条件。

技术分析：使用简单的移动平均线作为买入信号。
